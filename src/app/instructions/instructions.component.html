Introduction

Finding failure causes in build logs can be a struggle. Most build logs are several hundred (or even thousand) lines long and contain a lot of information that is not relevant. Also comparing two logs is not easy, as normal differencing tools cannot properly handle irrelevant information (e.g. download speeds, timestamps) that change from execution to execution. BLogDiff allows you to easily compare two logs and reduce the lines to be considered to a bare minimum. BLogDiff currently only works with Java projects build with Maven and Travis-CI.

Start Differencing

You have to options to start the differencing of two build logs. You can either use the manual mode, or directly start the differencing out of Travis-CI using the provided Google Chrome extension. (If you do not have the extension installed already, please click here)

Manual Mode

When using the manual mode, you have two options. You can either user you "repository slug" or manually enter two job ID’s.

The repository slug can be inferred from the Travis-CI or the Github URL of your project and consists out of your username and the repository name. After submitting, a list of jobs will be loaded (see Select Jobs ).

Example

Travis-CI URL: https://travis-ci.org/noahch/blogdiff -> Repository Slug: noahch/blogdiff
Github URL: https://github.com/noahch/blogdiff -> Repository Slug: noahch/blogdiff

You can also just enter two job ID’s and submit the request. You will find the job ID in the first few lines of a Travis-CI build log.

Attention

Ensure that bot job ID's are from the same project.

Google Chrome Extension

Once you have installed the extension, you will find this icon in all your projects. Simply click on the icon and you will be redirected to the BLogDiff Application.

Select Jobs

Once you have selected a repository via extension or manual mode, two list of jobs are displayed. You can directly see the status of the build on the left side as well as further detailed information. After submitting the request, the differencing result will be displayed.

Info

The list of jobs only contains the last 20 jobs. If you want to compare jobs that are older, please use the manual mode.

Basics

Once the differencing is done, you can compare the two logs. The top row shows you basic statistics on the amount of changes. You can also collapse individual modules to get a better overview. Additions of lines are highlighted in green, deletions of lines in red, updates of lines in yellow and moves of lines in purple.

Settings

You have several settings, which should help you to extract the information you need more quickly.

  • Show Additions: Enable or disable the highlighting of additions of lines
  • Show Deletions: Enable or disable the highlighting of deletions of lines
  • Show Updates: Enable or disable the highlighting of updates of lines
  • Show Moves: Enable or disable the highlighting of moves of lines
  • Highlight Moves: Enable or disable the highlighting of moved lines on click. When enabled and clicked on a move, the according lines in the other log are highlighted.
  • Wrap Lines: Enable or disable the wrapping of lines
  • Differences Only: When enabled, only lines that have changed are shown.
  • Symmetric Nodes: When enabled, only nodes (e.g. Maven modules) that are present in both logs are shown.

Tips

  • The most efficient way is to enable "Differences Only" and only highlight updates.
  • If that is not enough, also enable additions and deletions.
  • If you have any problems or the tool does not work as expected, please file a bug report (right side in the settings panel).

Warning

BLogDiff currently only works with Maven projects in combination with Travis-CI.
